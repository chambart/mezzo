Module Permissions

module Permissions: sig .. end
This module provides permission manipulation functions.
val collect : Types.typ -> Types.typ * Types.typ list
collect t recursively walks down a type with kind TYPE, extracts all the permissions that appear into it (as tuple or record components), and returns the type without permissions as well as a list of types with kind PERM, which represents all the permissions that were just extracted.
val unfold : Types.env -> ?hint:Types.name -> Types.typ -> Types.env * Types.typ
unfold env t returns env, t where t has been unfolded, which potentially led us into adding new points to env. The hint serves when making up names for intermediary variables.
val unify : Types.env -> Types.point -> Types.point -> Types.env
unify env p1 p2 merges two points, and takes care of dealing with how the permissions should be merged.
val add : Types.env -> Types.point -> Types.typ -> Types.env
add env point t adds t to the list of permissions for p, performing all the necessary legwork.
val add_perm : Types.env -> Types.typ -> Types.env
add_perm env t adds a type t with kind PERM to env, returning the new environment.
val sub : Types.env -> Types.point -> Types.typ -> Types.env option
sub env point t tries to extract t from the available permissions for point and returns, if successful, the resulting environment.
val sub_perm : Types.env -> Types.typ -> Types.env option
sub_perm env t takes a type t with kind PERM, and tries to return the environment without the corresponding permission.
val sub_type : Types.env -> Types.typ -> Types.typ -> Types.env option
sub_type env t1 t2 tries to perform t1 - t2. It is up to the caller to "do the right thing" by not discarding t1 if it was not duplicable. Unifications may be performed, hence the return environment.
val fold : Types.env -> Types.point -> Types.typ option
fold env point tries to find (hopefully) one "main" type for point, by folding back its "main" type t into a form that's suitable for one thing, and one thing only: printing.
