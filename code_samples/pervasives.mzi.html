 7 (* Options *)
 8 
 9 data option a = None | Some { contents: a }
10 
11 data choice a b =
12   | Left  { contents: a }
13   | Right { contents: b }
14 
15 (* Interesting. You cannot put “val none : [a] option a”: although this
16  * signature will be accepted, consumers will no longer know that [none] is
17  * duplicable. Therefore, the first consumer that uses [none] will consume the
18  * permission for it, which is not what we want.
19  *
20  * A first idea would be to use an argument in the style of the value
21  * restriction: since [a] only appears in positive position in “[a] option a”,
22  * it means [none] does not use its parameter. Type [option] is not declared as
23  * exclusive, so [none] *must be* duplicable.
24  *
25  * A second idea would be to add a mechanism to mark a value in a signature as
26  * duplicable.
27  *
28  * A third idea would be, assuming we want the type [option] to be abstract, to
29  * write “val none : [a] () -> [option a]”. The implementation would be
30  * “val none [a] (): option a = None” (the constant can even be shared).
31  *
32  * Since we won't make option abstract, we can afford to put the real
33  * constructor here. *)
34 val none : None
35 val some : [a] (consumes a) -> option a
